TR1 specifies new components. For exxample

boost::shared_ptr

Here are some of the TR1 components

Smart pointers are objects acting like pointers but avoids some of the problems often encountered with pointers. A smart pointer contains a native pointer as a data member and provides a set of overloaded operators that make it act like a pointer in most ways. The pointers can be dereferenced, so * and -> still work. Pointer arithmetic also still works. A smart pointer is an object, and it contains additional meta data and member and behavior. For example a smart pointer can recognize when it has been deleted, returning NULL when this happens.

Smart pointers also facilitates reference counting. If the pointer reference drops to zero, the object is no longer needed and can automatically be deleted. This saves you from having to worry about ownership and orphaned objects.

There is also the shared_ptr. When copying an STL auto_ptr using the copy constructor for example, ownership is transferred from that being copied to that being copied to. The pointer being copied will also be set to NULL. We can replace auto_ptr with share_ptr
